4. Key Security ConsiderationsCustomer Creation (KYC)Field Agent Limitations: On a TPE, agents often register customers in offline areas. Use create_prospect so the customer profile is "Pending" until a back-office user verifies the uploaded ID documents.Sensitive Edits: Changing a customer's registered phone number (used for Mobile Money) is a high-risk action. Restrict this to the supervisor role only.Disbursement SafetyDaily Disbursement Limits: Even if an agent has the request permission, you might want to hard-code a limit in the mutation. For example, any disbursement over 500,000 CFA might require two supervisors to approve.TPE Specificity: For physical cash-outs at the TPE, the software should verify that the device has a high enough "Float Balance" before allow a disbursement request to be initiated.5.
